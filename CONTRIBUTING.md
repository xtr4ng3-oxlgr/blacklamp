# Contribuir

BLACKLAMP debe mantenerse simple, local y defensivo.

## Reglas

- No agregar telemetría.
- No agregar borrado automático.
- No agregar funciones ofensivas.
- No agregar monitoreo oculto.
- No modificar el sistema sin confirmación.
- Explicar hallazgos de forma clara.
- Priorizar seguridad del usuario común.

## Mejoras aceptables

- Reglas anti-phishing más claras.
- Mejor análisis de dobles extensiones.
- Reportes más legibles.
- Pruebas.
- Mejoras visuales.
- Documentación útil.
